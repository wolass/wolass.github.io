Init lab

Fri, Jul 14, 2017 scientific communication blog, github, science

Introduction

This is my initial post. Communication in science is changing not so rapidly as it should. Use of new tools and methods may drive change in the way we communicate experiment results. I have been using R and Rstudio extensively since last 4 years. It has proven exeptionally well suited for making science and my latest attempts at producing full manuscripts in RStudio were suprisingly successful. This lead me to investigating new forms of scientific communication.

Aim

The purpose of this site is to a) order my knowledge and programming skills, b) make me practice clear communication with the readers, c) promote Open Science and d) serve as an Open Lab Notebook.

Methods

I will be using the ‘blogdown’ package by Yihui Xie to serve this site. My ideas and data will be shared openly using the CC-BY 4.0 License. There is no funding for this site, and the data I share here are my own. To host the page I’ll use github and AWS for the domain.

Results

So far using blogdown was a blast and it works great. I have a lot of topics to cover in the following posts. This will be documenting my experience how to set up an academig website using blogdown and github.

Setup

The complete instructions to get you started right away are written here by the author of the blogdown package. If you want th get right to business try this page.

In short: setting up the blogdown and hugo and previewing locally takes no more than 5 minutes!

The time consumig part is getting all the personal details and the list of publications into the file structure.

I really love that everything here is written in markdown!

Academic details

I chose the academic template because it ideally answers my needs. You need to add some links to the most used account in the configuration file: Research gate, Impactstory, twitter links and so on. It takes a couple of hours. To represent your current CV.

Publishing

I already had a domain and used the github hosting service. So for me this did not make a difference, but setting these up is also not that hard. Details are in the Blogdown’s manual.

Discussion

Could this type of content be the future of scientific communication? Or are we bound to paper journals and good ol’ Impact Factor?
